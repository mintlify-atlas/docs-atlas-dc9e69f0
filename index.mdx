XLA (Accelerated Linear Algebra) is an open-source machine learning compiler built collaboratively by Alibaba, Amazon Web Services, AMD, Apple, Arm, Google, Intel, Meta, and NVIDIA as part of the [OpenXLA](https://openxla.org) project.

XLA takes ML models expressed in [StableHLO](https://github.com/openxla/stablehlo) — the standard portability layer from frameworks like JAX, PyTorch, and TensorFlow — and compiles them into optimized machine code for your target hardware.

## What XLA does

XLA sits between ML frameworks and hardware, compiling high-level operations into efficient machine code:

1. **Accepts StableHLO** — the versioned, portable ML operation set from JAX, PyTorch, and TensorFlow
2. **Runs optimization passes** — algebraic simplification, operation fusion, rematerialization, and hundreds more
3. **Generates target code** — via LLVM IR for CPUs and NVIDIA/AMD GPUs, or via Triton for GPU kernels
4. **Executes via PJRT** — the uniform device runtime API used by all ML frameworks

## Framework integrations

XLA is already integrated into major ML frameworks. If you want to use XLA to compile your models without contributing to the compiler itself, refer to your framework's documentation:

<CardGroup cols={3}>
  <Card title="PyTorch/XLA" icon="fire-flame-curved" href="https://pytorch.org/xla">
    Use XLA as a PyTorch backend via torch_xla.
  </Card>
  <Card title="TensorFlow/XLA" icon="bolt" href="https://www.tensorflow.org/xla">
    Enable XLA JIT compilation in TensorFlow with @tf.function(jit_compile=True).
  </Card>
  <Card title="JAX" icon="atom" href="https://jax.readthedocs.io/en/latest/notebooks/quickstart.html">
    JAX uses XLA by default for all compiled computations via jax.jit.
  </Card>
</CardGroup>
