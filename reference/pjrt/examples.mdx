## Example: JAX CUDA plugin

The JAX CUDA plugin is the canonical example of a full PJRT C API integration.

<Steps>
  <Step title="Implement the PJRT C API">
    Provide a C API implementation through the GPU wrapper:
    [`pjrt_c_api_gpu.h`](https://github.com/openxla/xla/blob/main/xla/pjrt/c/pjrt_c_api_gpu.h)
  </Step>
  <Step title="Set up the package entry point">
    Define the Python package with an entry point so JAX can discover the plugin:
    [`setup.py`](https://github.com/google/jax/blob/main/jax_plugins/cuda/setup.py)
  </Step>
  <Step title="Implement initialize()">
    Register the plugin with `xla_bridge.register_plugin` in your module's `initialize()` method:
    [`__init__.py`](https://github.com/google/jax/blob/a10854786b6d1bc92a65dd314916b151640789af/plugins/cuda/__init__.py#L31-L51)
  </Step>
  <Step title="Test with JAX CUDA tests">
    The plugin can be tested with any JAX tests that target CUDA.
  </Step>
</Steps>

## Integrating a plugin with JAX

### Installing JAX

Use JAX nightly or build from source:

```bash
# Install JAX nightly
pip install --pre -U jaxlib -i https://us-python.pkg.dev/ml-oss-artifacts-published/jax/simple/
pip install git+https://github.com/google/jax

# Match jaxlib version with your PJRT C API version
pip install --pre -U jaxlib==0.6.1.dev20250428 -i https://us-python.pkg.dev/ml-oss-artifacts-published/jax/simple/
```

### Plugin discovery

JAX supports two methods for plugins to be discovered:

<Tabs>
  <Tab title="Namespace packages">

Place your plugin module under the `jax_plugins` namespace package:

```
jax_plugins/
  my_plugin/
    __init__.py
    my_plugin.so
```

  </Tab>
  <Tab title="Package entry points">

Advertise your plugin via `pyproject.toml` or `setup.py`:

```toml
# pyproject.toml
[project.entry-points.'jax_plugins']
my_plugin = 'my_plugin'
```

```python
# setup.py
entry_points={
  "jax_plugins": [
    "my_plugin = my_plugin",
  ],
}
```

  </Tab>
</Tabs>

### Registering the plugin

Implement `initialize()` in your Python module:

```python
import os
import jax._src.xla_bridge as xb

def initialize():
    path = os.path.join(os.path.dirname(__file__), 'my_plugin.so')
    xb.register_plugin('my_plugin', priority=500, library_path=path, options=None)
```

To verify the plugin loads and set it as the default backend:

```python
jax.config.update("jax_platforms", "my_plugin")
```

### Basic tests

```python
# Basic addition
print(jax.numpy.add(1, 1))
# => 2

# JIT-compiled function
print(jax.jit(lambda x: x * 2)(1.))
# => 2.0

# Multi-device pmap
arr = jax.numpy.arange(jax.device_count())
print(jax.pmap(lambda x: x + jax.lax.psum(x, 'i'), axis_name='i')(arr))
# single device: [0]
# 4 devices: [6 7 8 9]
```

## Implementing PJRT_Client_Create

The `PJRT_Client_Create` function is the main entry point for your plugin. Using the
C++ wrapper:

```cpp
#include "xla/pjrt/c/pjrt_c_api_wrapper_impl.h"

namespace my_plugin {

PJRT_Error* PJRT_Client_Create(PJRT_Client_Create_Args* args) {
  std::unique_ptr<xla::PjRtClient> client = GetPluginPjRtClient();
  args->client = pjrt::CreateWrapperClient(std::move(client));
  return nullptr;
}

}  // namespace my_plugin
```

Then expose `GetPjRtApi()`:

```cpp
const PJRT_Api* GetPjrtApi() {
  static const PJRT_Api pjrt_api =
      pjrt::CreatePjrtApi(my_plugin::PJRT_Client_Create);
  return &pjrt_api;
}
```

For a complete walkthrough see the [PJRT integration guide](https://github.com/openxla/xla/blob/main/xla/pjrt/c/docs/pjrt_integration_guide.md).
